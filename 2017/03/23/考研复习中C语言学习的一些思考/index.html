考研复习中C语言学习的一些思考

发表于 2017-03-23 | 分类于 若有所思

前言

一、我认为许多事情就是思想观念上的事情，毛主席讲，战略上要藐视敌人，战术上要重视敌人。我想对于C语言的学习也是这样，我所接触的一些考研要考C语言的同学，总能听到他们说一句：C语言好难啊，我学不会…还没有开始学，自己就把自己pass掉了，抱着这样的想法，怎么可能学得好呢？在开始学习它之前，首先应该扭转自己的偏见，之后正视它，认真对待它，学习它，这样就会事半功倍，否则事倍功半。

二、学习不能学的一团浆糊，不能说跟着课本过一遍之后，自己去回想好像似乎什么都知道一点，又似乎什么都想不起来。学习完C语言之后(第一遍看课本)，至少应该能清楚的讲明白，C语言可以分成几个部分，哪几部分比较简单，而哪几部分相对较难(第一遍学不易理解的)。这样清晰明了的学习，可以使我们增强自信，相信自己可以学好它，也使得再次学习也更有针对性。

理论

根据前言所述，思想观念转变之后，首先应该做的就是尽快弄清楚，C语言都有什么？这时候，我首先想到的是去看一本书的前言和目录，前言往往会对书的内容做一些简单介绍，而目录的各个章节就是这本书的基本骨架。这会给我留下一个最初的印象——我将要学习的都包括哪些内容。

C语言都包括什么呢(一门语言最基本的都有什么)？

基本数据类型

变量，从字面上来理解，就是一个可变的量，而在编程中，变量就是一个用于临时存储值的存储器，就像我们日常生活中需要将各种东西分门别类的整理，并放在一个个柜子里一样，在我们编程中总有各种类型的数据，如int型、char类型等等，这些数据需要不同的盒子去存储，C语言把数据存放在变量中，变量就是计算机内存里的一个盒子，用来保存保存特定类型的数据，而这个盒子中东西是可以改变的，例如我放一个数字1放进这个盒子，这个盒子现在装的是数字1，然后我又放数字2进入，原来的数字1就自动替换成数字2，然后你放什么进去，它就是总是替换，它是可变的，但需要注意的是，一种类型的盒子（变量）只能用于存放相应类型的数据，我们通过声明变量来创建盒子，通过变量名来标识盒子。

注: 注意变量名的命名规则

函数

C语言函数传参无论什么类型都是值传递，也就是说，函数参数是实际传入参数的复制或拷贝，而不是其本身，此时应注意普通类型变量和引用类型变量传参的区别

注：对于一个带有参数的函数，首先应当考虑所传参数的合法性，不能想当然的认为方法的使用者，总会传入正确的合法的值，应当考虑到当方法的使用者传入非法参数时的处理措施

指针

指针，就是存储单元的地址。一个变量的地址称为该变量的指针，即指针变量的值就是一个内存地址
我们之前了解到，变量可以看做一个盒子，用于存储不同的数据，指针变量与之相同，也可以看做是一个盒子，只不过普通变量是用于存储需要的数据如一个整数、一个字符等等，而指针变量用于存储那些普通变量在内存中的地址

指针是一个地址，而指针变量是存放地址的变量

应当注意，不同类型的数据在内存中所占字节数和存放方式是不同的，故在定义指针变量时，必须指定其基本数据类型，另外，指针变量中只能存放地址，就像一个整形变量只能存放整型数值一样，不能将一个常量赋值给一个指针变量

实践

编程语言的学习，最终都是要回到实际动手操作上来的，毕竟不是说只要把各个知识点都记住了，我们就能够写程序的。无论你前边理论学习的多好，不能写，不能实际的用来解决问题，都是白扯，实践很重要！正所谓实践是检验真理的唯一标准，我们前边学习了、理解了，但是自己理解的对不对？只有动手写程序，让程序跑起来，才能够检验和证明。

当然，就像我们从蹒跚学步，到如今的健步如飞，任何事情都需要一个循序渐进的过程。我们学习也是如此，C语言是面向过程的语言，面向过程，我自己的理解就是，它模拟我们日常的行为习惯，也就是说，你怎么想的，就怎么做，就可以了，而最终你会发现，我们所写的代码都是由一个个函数构成的，而函数又都是由顺序、分支、循环这三个结构构成的。下面通过一个简单的例子来开始我们的编程之路，并实际的理解面向过程的思想：

思考：如何把一头大象装进冰箱呢？

看到这个问题，我们应该想到什么？大象怎么能装进冰箱呢？！！！如果这么想，那我们就什么都做不了。如果你非要这么想，那么我们可以再次假设，怎么把一瓶水装进冰箱呢？首先、打开冰箱门，第二、把一瓶水放进冰箱，第三、关闭冰箱门，我们大脑中是这么想的，现实中，我们也是如此做的，这一系列操作就是一个过程。那么回到我们的问题上来，如何把一头大象装进冰箱呢？我想这个时候应该就都知道如何做了。

不要立即动手写代码

根据我们的想法，我们写了三个函数，就解决了这个问题，之后我们只需要在main()函数中分别调用这三个函数，你就可以把任意一头大象装进冰箱，想一想，如此简单的，你就把一头大象，装进了冰箱！！！

于此类似的，在我们动手开始写代码之前，都应该认真思考，我们应该怎么做才能解决？将一个大的问题进行分解，顺序的分为1、2、3、…n个步骤或者多个小问题，一个一个的解决它

回到课本

当我们动手写代码时，在使用某些知识时，如果不能明确，一定要定位到课本的相应位置，重新复习，要利用好课本的目录，以快速查找知识点。

在最后的复习阶段，我想也应该重新回到课本，重新梳理各章节知识点，查漏补缺。
