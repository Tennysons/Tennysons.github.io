数组初始化

java中数组在使用前必须先进性初始化，其初始化过程就是对数组的元素进行内存分配，同时指定其初始值得过程，其初始化数组的元素可以是任意类型，但必须与声明数组时的类型相同。在数组初始化完成后，不能够重新改编数组对象在内存中的位置和大小

数组声明及初始化的方式

1、数组的声明

数组的声明仅仅定义了数组名，其为引用类型变量，事实上此时仅在栈内存中定义了一个引用变量(即一个指针), 该变量未指向任何有效内存区(一个空引用)

2、数组初始化的第一种方式：静态初始化

数组静态初始化的方式是初始化数组与给数组元素赋值同时进行，即显示的指定每个数组元素的初始值

3、数组初始化的第二种方式：动态初始化

数组动态初始化的方式是初始化数组与给数组元素赋值分别进行，即初始化时只定义数组长度，系统按照其数据类型自动指定其每个数组元素的初始值（默认值）

注意：

Java中的数组变量时引用类型变量（其存储在栈内存中，在数组声明过程中产生，保存数组对象在堆内存中的首地址），数组变量并不是数组本省，只是一个指向对内存中的数组对象，故可以人为的改变其引用，数组变量不需要进行初始化，需要初始化的是数组对象（数组元素，其存在于堆内存中，为地址连续的内存空间） ，在初始化过程中需为其分配内存空间并赋初始值; Java中所有的引用变量都不需要初始化，该引用变量所引用的对象需要初始化
引用变量本质上只是一个指针，只要程序通过引用变量访问属性，或者通过引用变量调用方法，那么该变量将会由它所引用的对象所替代,当通过引用变量来访问属性，或者调用静态方法时，若该引用变量还未引用一个有效对象，程序就会引发NullPointerException(空指针异常)

4、动态初始化时各类型数组元素默认值

byte、short、int、long： 默认值为 0
float、double： 默认值为 0.0
char：默认值为 ‘\u000’
boolean：默认值为 false
引用类型（类、接口、数组）： 默认值为 null

[java数组内存分配]

5、将数组传递给方法

java中传参为按值传递，故对于基本数据类型参数，传递的是实参的值；
但对于数组类型的参数，参数值是数组的引用，因此在给方法传参时，传递的是数组的引用。（即参数传递的是共享信息， 在方法中的数组和传递的数组是一样的），故改变方法中的数组，外部数组也会改变
